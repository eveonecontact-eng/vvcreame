### ファイルの役割を理解するルール

- **アニメーションや動きを変えたい** → `main.js` を編集
- **新しいページを追加したい** → 既存の `.html` をコピーして作成し、全ページのナビゲーションにリンクを追加

## 4. デザインルール（編集時に必ず守ること）

### 配色
CSS変数で一元管理しています。`style.css` の `:root` 内で定義。

| 用途 | 変数名 | 値 | 注意 |
|------|--------|-----|------|
| テキスト | `--color-text` | `#45433E` | **純黒(#000)は使わない。** 温かみのあるダークブラウン |
| テキスト薄 | `--color-text-light` | `#6B6560` | 補足テキスト・日付などに使用 |
| 背景メイン | `--color-bg` | `#FFFFFF` | 白 |
| 背景サブ | `--color-bg-muted` | `#F1F1F1` | セクションを交互に切り替えて使う |
| アクセント | `--color-accent` | `#C80421` | **使いすぎ厳禁。** プログレスバー・特別なボタンにのみ |
| フッター背景 | `--color-footer-bg` | `#3D3D3D` | ダークグレー |

### フォント
- **英語の見出し：** `Cormorant Garamond`（セリフ体、大きめ、letter-spacing: 0.06em）
- **日本語の本文：** `Noto Sans JP`（ゴシック体、細〜標準ウェイト）
- **見出しの構造：** 英語大（3.4rem）→ 日本語小（1.2rem）の二段構成で統一
  ```html
  <h2 class="section-title-en">PRODUCTS</h2>
  <p class="section-title-ja">商品紹介</p>
  ```

### 余白
- セクション間の余白：`padding: 90px 0`（**これが高級感の根幹。絶対に詰めない**）
- コンテンツ最大幅：`1110px`
- 左右余白：`40px`（モバイル時 `20px`）

### ボタン
- 基本スタイル：アウトライン（背景透明＋細い枠線 `1px solid #45433E`）
- ホバー：背景と文字色が反転（白→黒、黒→白）
- 角丸：`border-radius: 4px`（丸すぎない上品さ）
- CSSクラス：
  - `.btn-outline` — 通常のアウトラインボタン
  - `.btn-outline-white` — ヒーロー上の白枠ボタン
  - `.btn-filled` — 塗りつぶしボタン（ECサイトリンク等、アクセントカラー）
  - `.btn-line` — LINE用の緑ボタン

### 画像
- すべて `assets/images/` フォルダに格納
- ファイル名形式：`LINE_ALBUM_20260314_260411_番号.jpg`
- 新しい画像も同フォルダに追加して、HTMLの `src` 属性を書き換える
- `loading="lazy"` を付ける（ファーストビューの画像のみ `loading="eager"`）

## 6. やりたいこと別の編集手順

### 画像を差し替えたい
1. 新しい画像を `assets/images/` に追加する
2. 該当HTMLファイルの `<img src="...">` のパスを新ファイル名に変更
3. `alt` 属性も内容に合わせて更新する

**AIへの指示例：**
```
ヒーローの1枚目の画像を new-hero.jpg に変えて。
altテキストは「新商品メインビジュアル」にして。
```

### 商品を追加したい
1. `index.html` の `products-grid` 内に `.product-card` ブロックをコピーして追加
2. 商品名・説明・画像・リンク先を書き換える
3. 必要なら `products.html` を複製して新商品の詳細ページも作成
4. 全ページのナビゲーションに新ページへのリンクを追加

**AIへの指示例：**
```
index.htmlのPRODUCTSセクションに新しい商品カードを追加して。
商品名は「モイスチャーセラム」、説明は「高保湿美容液」、
画像は LINE_ALBUM_20260314_260411_50.jpg を使って、
リンク先は products-serum.html にして。
```

### お知らせを追加したい
`news-list` 内に以下の形式で `news-item` を追加：
```html
<li class="news-item">
  <time class="news-date" datetime="2026-05-01">2026.05.01</time>
  <span class="news-tag">NEW</span>
  <a href="#" class="news-title">タイトル</a>
</li>
```

**AIへの指示例：**
```
NEWSに追加して。日付は2026年5月1日、タグはNEW、
タイトルは「新商品モイスチャーセラムを発売しました」
```

### URLを設定・変更したい
現在 `href="#"` になっている箇所を実際のURLに差し替える。

| 対象 | ファイル | 場所 |
|------|---------|------|
| ECサイト | `index.html` | CTAセクションのボタン |
| ECサイト | `products.html` | 「ECサイトで購入する」ボタン |
| Instagram | `index.html` | SNSセクションのリンク |
| 公式LINE | `index.html` | SNSセクションのリンク |
| 公式LINE | `contact.html` | 「LINEで問い合わせる」ボタン |

**AIへの指示例：**
```
ECサイトのURLを https://example-shop.com に設定して。
index.html と products.html の両方を変更して。
```

### Googleフォームを埋め込みたい
1. `contact.html` を開く
2. `form-placeholder` の `<div>` を削除する
3. コメントアウトされている `<iframe>` のコメントを外す
4. `src="YOUR_GOOGLE_FORM_URL"` を実際のURLに書き換える

**AIへの指示例：**
```
contact.html にGoogleフォームを埋め込んで。
URLは https://docs.google.com/forms/d/e/XXXXX/viewform
プレースホルダーは削除して。
```

### 新しいページを追加したい
1. 既存のHTMLファイル（例：`about.html`）をコピーして新ファイルを作成
2. ページタイトル・内容を書き換える
3. **全ページ**（index/about/products/contact＋新ページ）のナビゲーションにリンクを追加
   - ヘッダーの `.nav-list`（フルスクリーンナビ）
   - フッターの `.footer-nav`

### ブログ記事を追加したい
`products.html` の構成を参考に、新しいHTMLファイルとして作成する。
またはNEWSセクションの各お知らせにリンク先として個別ページを用意する。

## 8. デプロイ方法

### 編集後の公開手順
```bash
# 1. 変更をステージング
git add .

# 2. コミット（変更内容を簡潔に書く）
git commit -m "トップページのヒーロー画像を差し替え"

# 3. GitHubにpush（自動でCloudflareにデプロイされる）
git push
```

### 手動デプロイ（GitHub連携がない場合）
```bash
wrangler pages deploy . --project-name client-c
```

- またはVSCodeのLive Serverを使用
- **注意：** Swiper.jsはCDN経由のため、オフライン環境ではカルーセルが動かない

## 9. 編集時の注意事項

### やってはいけないこと
- `style.css` の `--color-*` 変数を無断で変更しない（全ページのデザインが崩れる）
- セクション間の `padding: 90px` を小さくしない（高級感がなくなる）
- 純黒 `#000000` をテキストに使わない（`#45433E` を使う）
- `assets/images/` 以外の場所に画像を置かない
- ナビゲーションリンクを1ページだけ変更しない（**全ページで統一する**）

### 気をつけること
- 画像を追加したら `alt` 属性を必ず設定する
- 新しいセクションには `reveal` クラスを付けるとスクロール時にフェードインする
- レスポンシブ対応：`768px` 以下でレイアウトが変わるので、モバイル表示も確認する
- 全ページで共通のヘッダー・フッターを使っているので、変更時は全ファイルに反映する

## 10. 困ったときの対処法

### 「デザインが崩れた」
→ `git log` で直近の変更を確認し、`git checkout -- ファイル名` で元に戻す

### 「画像が表示されない」
→ `src` のパスが正しいか確認。`assets/images/` 内にファイルがあるかを `ls` で確認

### 「pushできない」
→ `git pull` でリモートの変更を取り込んでから再度push

### 「Swiperが動かない」
→ インターネット接続を確認（CDN経由で読み込んでいるため）
